# Step7 测试蓝图输出模板

Step7 的目标是把测试对象、subsection、适用维度、固定块和资源前提整理成稳定蓝图，为 Step8 的组合展开提供唯一依据。

## 固定结构

输出文件 `step7_test_blueprint.md` 必须包含以下二级标题，且顺序固定：

```md
# Step7 测试蓝图
## 规则装载
## 测试对象总览
## Subsection 蓝图
## 适用维度
## 资源类型映射
## 固定块与必测项
## 资源前提
## 黄金参考继承策略
## 排除项
## 来源映射
```

## 写法要求

- `规则装载`：列出本次实际加载的 common/product/knowledge/resource/golden 规则，并说明主产品与补充产品。
- `测试对象总览`：列出本次任务下需要保留的对象树，例如对象、页面动作、相关影响块。
- `Subsection 蓝图`：逐对象列出其 subsection，不要把多个页面或多个对象混在一条里。
- `适用维度`：必须写清“哪个对象 / 哪个 subsection 适用哪些维度轴”。
- `资源类型映射`：必须写清“哪个对象 / 哪个 subsection 依赖哪些资源类型”。
- `固定块与必测项`：显式列出事件审计、权限管理、异常限制、VPC 影响项等不得遗漏的块。
- `资源前提`：写出 Step8 组合展开前必须显式保留的资源模式、环境对象和拓扑前提。
- `黄金参考继承策略`：若采用黄金参考，说明继承哪些结构；若未采用，明确写跳过原因。
- `排除项`：说明哪些内容虽然在参考资料里出现，但当前需求不纳入。
- `来源映射`：把蓝图中的主要对象、subsection 和维度映射回需求文档、图片、历史样本或 SmartX docs。

## 目标

- 蓝图必须足够清晰，让 Step8 可以直接展开组合而不再猜“哪些维度适用于哪些 subsection”。
- 蓝图必须显式承接 Step6 的产品对象和术语校准结论，以及产品知识摘要中的对象树和高优先级拆分轴。
- 若采用黄金参考，蓝图应向黄金参考对齐结构和覆盖；若未采用，则按当前材料自行组织。
- 蓝图应提前标记哪些位置存在复合 case 风险，以便 Step8/Step10 继续细分。

## 禁止项

- 禁止把 `适用维度` 写成与对象无关的全局散点清单。
- 禁止把 `资源类型映射` 漏掉后又在 Step8 临时补猜。
- 禁止遗漏固定块后再指望 Step9 补回来。
- 禁止在 Step7 直接落组合或 `[C]` testcase。
- 禁止直接照搬历史样本中的复合 case 标题，而不补足中间拆分层。
