# Step9 最终测试用例输出模板

Step9 的最终用例输出是树形缩进列表，不使用 markdown `#` 标题。

## 输出目标

- 每一行都必须以 `- ` 开头。
- 通过四个空格表示一层缩进，不使用两个空格、Tab 或其他缩进宽度。
- 分组节点禁止带 `[C] ` 前缀。
- 具体测试用例节点必须带 `[C] ` 前缀。
- 每个节点必须紧跟 3 行子项：`描述：`、`来源：`、`承接：`。
- `承接：` 必须写明当前节点承接自 Step8 的哪个组合节点。

## 最小示意

```md
- 某分组节点
    - 描述：说明该分组承接的对象、页面动作或校验维度。
    - 来源：通过《某需求.md》中的“某规则”内容，结合某 png 中的“某页面区域”内容得出。
    - 承接：承接自 Step8 组合展开中的“某组合分组”。
    - [C] 某具体测试用例
        - 描述：验证某对象在某条件和资源前提下执行某动作时结果符合预期。
        - 来源：通过《某需求.md》中的“某限制条件”内容，结合某 png 中的“某控件/某提示”内容得出。
        - 承接：承接自 Step8 组合展开中的“某组合叶子”。
```

## 硬性约束

- 所有非空行都必须是树形列表项。
- 只允许使用四个空格做缩进。
- 每个节点后必须立刻跟 `描述：`、`来源：`、`承接：` 三行。
- `[C]` 只能落在最细层。
- 不允许把“不同版本 / 不同权限 / 不同主备关系 / 不同结果 / 不同资源模式”压成一条 case。
- 不写编号，不写表格，不写“同上”“略”“待补充”。

## 对齐目标

- 若采用黄金参考，则结构和覆盖优先向黄金参考对齐。
- 若未采用黄金参考，则以当前需求材料定义的对象树、subsection、固定块和资源类型为准，不设固定数量目标。
