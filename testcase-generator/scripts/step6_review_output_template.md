# Step6 Review 模板

主 skill 在完成汇总后，必须输出 review markdown，结构固定如下：

```md
# Step6 Review

## 汇总结论

- 说明本次 review 回看了哪些原始材料，以及比对了哪些 subagent / merged 产物
- 给出对 merged 版本的整体判断

## 发现与修正

- 如无问题，写 `- 无`
- 如有问题，按“问题 -> 修正”的形式逐条记录

## 最终判断

（通过）
```

## 约束

- 只允许使用标题、项目符号和最后一行括号结论
- `## 最终判断` 下只能填写 `（通过）` 或 `（需补充）`
- review 文件只记录复核结论与修正动作，不重复抄写全部测试用例正文
