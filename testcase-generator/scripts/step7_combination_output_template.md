# Step7 组合展开输出模板

默认的组合展开结果使用“树形缩进列表”，不是 markdown `#` 标题。

## 输出目标

- 每一行都必须以 `- ` 开头。
- 通过四个空格表示一层缩进，不使用两个空格、Tab 或其他缩进宽度。
- 层级无限，不限制只能写到几层。
- Step7 的输出禁止出现 `[C] ` 前缀。
- `描述：` 和 `来源：` 必须写成带 `- ` 的独立行，并且相对于所属节点多缩进一层。

## 层级规则

- 先按场景或对象分组，再按主变量轴继续分组。
- 主变量轴通常是许可子类、版本、环境对象、容量状态、权限状态、结果归属或联动关系。
- 如果一个节点下仍存在明显可拆分的变量轴，就必须继续分组，不要过早收束。
- 同一层内颗粒度必须一致，不要把分组节点和组合结论混在同一层。

## 最小示意

```md
- 某场景分组
    - 描述：说明该场景承接的主要测试空间。
    - 来源：通过《某文档.md》中的“某规则”内容，结合某截图中的“某页面区域”内容得出。
    - 某变量轴分组
        - 描述：说明该层表示的许可子类、版本、环境对象、容量状态或结果方向。
        - 来源：通过《某文档.md》中的“某规则”内容，结合某截图中的“某控件/某文案”内容得出。
```

## 硬性约束

- 所有非空行都必须是树形列表项，必须以 `- ` 开头。
- 只允许使用四个空格做层级缩进。
- 每个节点必须紧跟两行子项：`描述：`、`来源：`。
- `描述：` 和 `来源：` 的缩进必须比所属节点多四个空格。
- 一个节点写完 `描述：` 和 `来源：` 后，才可以继续写它的子节点。
- Step7 输出中禁止出现 `[C] ` 前缀。
- 不写编号，例如 `1.`、`TC001`、`case-01`。
- 不写表格，不写游离段落，不写“同上”“略”“待补充”。
