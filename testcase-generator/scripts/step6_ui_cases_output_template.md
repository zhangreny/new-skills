# Step6 用例输出模板

默认的 UI 前端测试用例 markdown 结构

## 默认结构

只使用 markdown 标题层级组织结构，注意标题不限于二层，可延伸至无数层。假设标题延伸至 N 层，用例将位于 N+1 层即可

```md
# 一级标题
描述：
来源：
## 二级标题
描述：
来源：
### 用例名称
描述：
来源：
```

## 约束

- 标题和用例的格式均必须是“用例名称 + 空行 + 下一行固定 `描述：` + 下一行固定 `来源：`”
- 不写编号
- 不写表格
- 不写多余章节
- 描述的内容是本条用例具体测试什么内容
- 来源的内容要交代清楚从哪个 markdown 文件或者结合哪张截图分析得出需要编写本条用例
