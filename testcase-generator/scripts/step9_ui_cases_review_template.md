# Step9 用例复核模板

输出文件 `step9_ui_cases_review.md` 必须包含以下二级标题，且顺序固定：

```md
# Step9 用例复核
## 覆盖结论
## 去重与拆分原则
## 固定块检查
## 主要风险与缺口
```

## 写法要求

- `覆盖结论`：总结最终用例覆盖了哪些对象、subsection 和维度轴。
- `去重与拆分原则`：说明哪些位置做了去重、哪些位置必须拆开，以及为什么。
- `固定块检查`：逐项确认事件审计、权限管理、异常限制、相关影响块是否保留。
- `主要风险与缺口`：如果还有结构或覆盖上的残余风险，必须写清楚；对于仍带有复合信号的 case，要写明会交由 Step10 继续细分。
