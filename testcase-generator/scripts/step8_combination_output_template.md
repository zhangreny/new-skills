# Step8 组合展开输出模板

Step8 的输出是树形缩进列表，不使用 markdown `#` 标题。

## 输出目标

- 每一行都必须以 `- ` 开头。
- 通过四个空格表示一层缩进，不使用两个空格、Tab 或其他缩进宽度。
- Step8 全文件禁止出现 `[C] ` 前缀。
- 每个节点必须紧跟 3 行子项：`描述：`、`来源：`、`承接：`。
- `承接：` 必须说明这个节点承接自 Step7 的哪个对象、subsection 或维度映射。

## 最小示意

```md
- 某组合分组
    - 描述：说明该节点承接的对象、subsection、主变量轴或组合目的。
    - 来源：通过《某需求.md》中的“某规则”内容，结合某 png 中的“某页面区域”内容得出。
    - 承接：承接自 Step7 测试蓝图中的“某对象 > 某 subsection > 某适用维度”。
    - 某组合叶子
        - 描述：说明该叶子已经是可执行组合，但仍未落成 testcase。
        - 来源：通过《某需求.md》中的“某限制条件”内容，结合某 png 中的“某控件/某提示”内容得出。
        - 承接：承接自 Step7 测试蓝图中的“某对象 > 某 subsection > 某资源前提”。
```

## 硬性约束

- 所有非空行都必须是树形列表项。
- 只允许使用四个空格做缩进。
- 每个节点后必须立刻跟 `描述：`、`来源：`、`承接：` 三行。
- 禁止 `[C]`、编号、表格、游离段落。
- 不允许把不同版本、不同权限、不同结果、不同主备关系提前合并。
- 如果一个节点仍然同时表达两个以上独立结果或两个以上独立资源模式，就必须继续拆开。
