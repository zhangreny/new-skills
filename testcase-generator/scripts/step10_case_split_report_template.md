# Step10 用例细分报告模板

输出文件 `step10_case_split_report.md` 必须包含以下二级标题，且顺序固定：

```md
# Step10 用例细分报告
## 轮次概览
## 数量闸门结果
## 命中的拆分信号
## 已拆分用例
## 未拆分原因
## 最终结论
```

## 写法要求

- `轮次概览`：说明本次读取了哪一份最终用例文件、当前 round_index、是否沿用 Step9 还是上一轮 Step10 产物。
- `数量闸门结果`：写明 `figma_total_bytes`、`target_min_case_count`、`current_case_count`、`numeric_gate_skipped`、是否达到数量目标。
- `命中的拆分信号`：列出当前仍存在的复合 case 信号，例如“同时/反复/复合/批量/多个动作/多个结果/多个资源模式”。
- `已拆分用例`：记录本轮实际拆开的 case，以及拆分后新增了哪些中间层或 `[C]`。
- `未拆分原因`：对于仍未拆开的 case，说明为什么暂时无法继续拆，或缺失了哪类证据。
- `最终结论`：说明本轮之后是否还需要继续 Step10，或是否因为内部保护而停止继续细分。

## 禁止项

- 禁止只写数量，不写具体命中的拆分信号。
- 禁止把“没有继续拆分”写成默认合理，而不解释原因。
- 禁止在报告里把“达到内部轮次上限”写成“已经达标”。
