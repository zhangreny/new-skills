# Step6 维度拆解输出模板

Step6 的输出不是树形结构，而是“每行一个维度”的确认清单。

## 输出目标

- 每个非空行都必须是一个维度。
- 每一行格式固定为：`xx维度：值1 / 值2 / 值3`。
- 使用中文全角冒号 `：`。
- `：` 后面的多个值必须通过 ` / ` 分隔。
- 不要使用 `- `、编号、表格、标题、代码块或其他标记。

## 写法要求

- 只输出可复用的分组维度，不输出 testcase 级结论。
- 维度名称应尽量稳定，便于 Step7 直接展开组合。
- 每一行都应该是“后续还能组合”的测试变量，不要直接写最终测试结果。
- 不要把多个不同维度混在同一行里。
- 只保留与当前材料直接相关的维度和值，不要为了凑完整性硬写无依据维度。

## 示例

```text
场景维度：卡片展示 / 集群扩容 / 部署 ER / 启用 ER / 创建 LBI / DFW 文案调整 / VPC 文案调整
关联功能维度：DFW / LB / VPC
许可子类维度：DFW-vm / DFW-cpu socket / LB-vCPU / LB-cpu socket / VPC-cpu socket
环境对象维度：新集群 / 旧集群 / LBI 所在集群 / 被 DFW 关联集群 / 未被 DFW 关联集群
容量状态维度：充足 / 刚好满足 / 不足 / 已满
结果维度：成功 / 报错 / 不报错 / 文案调整 / 许可增加 / 许可不变 / 许可归还
```
