# 背景

SMTX OS（ELF）或 SMTX ELF 集群关联到 Everoute VPC 后，将为集群创建「虚拟专有云系统网络」，作为集群系统网络的一部分，并为集群中每个主机分配对应的 TEP IP。

当前集群的 TEP IP 在分配后不支持修改，在集群整体搬迁的场景下，需要根据新机房的网络情况调整系统网络 IP，包括管理网络以及虚拟专有云系统网络，目前管理网络支持从界面编辑，TEP IP 也希望支持此功能。

同样，在集群搬迁的场景中，此集群中部署的系统服务虚拟机，对应的系统网络也需要随之调整，相关的包括：

* Everoute Controller 管理网络  
* 负载均衡虚拟机管理网络  
* 边缘网关虚拟机管理网络、TEP 网络

另外，目前分配 TEP IP 的流程中，如用户希望顺序指定所有主机中的 TEP IP 地址，缺少批量操作入口，因此操作上比较复杂，希望优化此配置流程。（[Slack 讨论](https://smartx1.slack.com/archives/C014SKH8823/p1721024246808299?thread_ts=1721024044.406799&cid=C014SKH8823)）

# 调整说明

## **TEP IP 指定方式**

* 关联集群 \- TEP IP 部分  
  * 展示顺序调整为按主机名称，升序排列；  
  * 支持批量填充 IP 地址；  
  * 如留空自动分配，也需要按照与界面相同的顺序依次分配。

  例如，集群中包含 4 个节点，node31、node32、node33、node34，TEP IP 池为 10.132.10.10-10.132.10.20，预期效果如下：

  ![][image1]

* 关联集群展示：按主机名称，升序排列，展示主机与对应的 TEP IP。  
* 旧版本（3.2 以下）中，如扩容后需为新节点增加配置 TEP IP，其他节点不允许编辑，此时编辑页面内批量填充部分不允许填写，仅支持对新主机填写 TEP IP。🟡 2025-03-10 补充限制

## **未指定 TEP IP 提示**

如集群扩容后，部分主机未指定 TEP IP，在概览页增加提示。

* 概览页提示 🟡 2025-03-10 补充预期展示效果  
  * 截图处增加一个「注意」类提示，告知用户及时为集群内新增主机添加 TEP IP。  
  * 调整后此处的提示包含两种异常情况  
    * ER 代理异常  
    * 主机未配置 TEP IP

    异常数字调整存在上述任何一种异常的主机数。

  * 点击后，弹窗中依次展示  
    * 以下主机的 ER 代理异常，xxx  
    * 以下主机未配置 TEP IP 地址，xxx

    ![][image2]![][image3]

* 报警提示：此场景下增加「信息」类告警，如集群中主机持续 1h 无 TEP IP 地址（为用户预留扩容后配置 TEP IP 的时间，不立即触发报警），提示用户尽快为集群中的新增主机添加 TEP IP。

| 报警字段 | 内容 |
| :---- | :---- |
| 报警文案 | Everoute 虚拟专有云网络已关联的集群 xx 中，主机 xx、xx 未添加 TEP IP 地址。 |
| 触发原因 | 集群扩容后，没有为新增主机配置 TEP IP 地址。 |
| 影响 | 集群中的虚拟机增加或删除 VPC 类型的虚拟网卡时可能会失败。 |
| 解决方法 | 将主机的网口绑定至虚拟分布式交换机 {{vds 名称}}，并为主机添加 TEP IP。 |
| 报警等级 | 信息 |
