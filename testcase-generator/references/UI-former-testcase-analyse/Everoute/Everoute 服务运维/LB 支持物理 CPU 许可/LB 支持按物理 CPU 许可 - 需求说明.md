# 背景

由于 Everoute 3.4 中，LB 支持了 VPC 功能，后续在企业云的版本里，也会将网络相关能力（包括防火墙、LB、VPC）打包组合，作为“云网络套件”销售。

因此希望云网络套件中的各种功能可以统一许可方式，**均按照集群数量及规模销售**，方便打包和计价，客户只需要根据企业云集群的物理资源数量，为每个集群购买全部网络功能，而无需单独考虑每个子功能下的资源用量。

Everoute LB 功能目前支持按照 LB 虚拟机使用的 vCPU 数量许可，即每创建一个 LB 实例，按照对应的 LB 虚拟机数量 \* 配置类型中的 vCPU 数量，消耗许可数。希望调整为支持按集群的物理 CPU 数量许可，有两个许可方向：

1. 按使用 LB 功能的集群许可  
2. **按 LB 虚拟机所在的集群许可**

由于目前 LB 功能支持的方式灵活，不仅可以对 SMTX 集群内的虚拟机提供负载均衡，也可以为其他位置的虚拟机、裸金属服务器、容器等提供服务，因此无法按照 1 的方式严格控制，我们也不希望将 LB 的适用范围只局限于 SMTX 集群内。因此将**采用方式 2，按照 LB 实例/ LB 虚拟机所在的集群许可**。

各功能支持的许可模式如下：

| 功能 | 按集群许可（节点的物理 CPU 插槽数） | 按 vm 许可 | 按 vCPU 数许可 |
| :---: | :---: | :---: | :---: |
| DFW | ✅ 按关联集群 | ✅ 按策略对象 | \- |
| VPC | ✅ 按关联集群 | \- | \- |
| LB | 此需求场景，支持按 LBI 所在集群许可 | \- | ✅ 按 LB 虚拟机使用的 vCPU 数 |

# 许可计算规则

## **计算方式**

每在 SMTX 集群中部署任一 LB 实例（实例中所有 LB vm 都在同一集群内），消耗的许可数 \= 集群内所有主机的物理 CPU 插槽总数。

说明：

* 许可消耗的**最小粒度为“集群”而非“主机”**。

  例如，在一个 3 节点集群中部署 LBI，其中包含 2 个 LB 虚拟机，分别部署在 2 个主机上，此时仍需要按照集群中所有节点数（即 3 节点）计算许可，后续此集群如扩容到 5 节点，即使新节点上未部署 LB 虚拟机，也需要跟随集群粒度，多消耗 2 节点的许可。

* 在一个集群中，不关心实际部署的 LB 虚拟机数量及 LBI 数量，部署多个时，不额外消耗许可。

## **消耗许可**

在如下操作中，需消耗对应数量的许可：

* 创建 LB 实例（部署 Everoute 时创建、启用 LB 功能时创建、单独手动创建）  
  * 如此 LB 实例所属集群未被消耗许可，则此次创建时消耗，创建时检查，如可用许可不足，则不允许创建。  
  * 如所属集群已消耗许可，即集群上已有其他 LB 实例，此次创建时不额外消耗。  
* 集群扩容  
  * 包含 LB 实例的 SMTX 集群，添加主机时，进行许可检查，如当前可用许可不足，则不允许扩容；  
  * 扩容后，按此次扩容节点的 CPU 插槽数消耗许可数。

## **归还许可**

在如下操作中，需归还对应数量的许可：

* 删除 LB 实例：删除后，如所在集群中无其他 LB 实例，则按集群节点的 CPU 插槽数归还许可。  
* 集群缩容（命令行操作）：缩容后，按减少的节点中 CPU 插槽数归还许可。

# 许可信息

修改后的许可信息中，包含的字段如下表所示：

| 字段 | 取值 | 说明 |
| :---- | :---- | :---- |
| **序列号** | Everoute 软件序列号 | 一个 Tower 中有且只有一个 ER 软件序列号 |
| **版本** | 标准版 | LB 暂无社区版 |
| **许可类型** | 试用许可/订阅许可/永久许可 | 许可类型之间可以相互转换 |
| **许可方式** | vCPU 数/CPU 插槽数 | 许可方式之间可以相互转换 |
| **许可额度** | \- | 按 vCPU 许可：CloudTower 中所有负载均衡虚拟机消耗的 vCPU 数量之和。 按 CPU 插槽数许可：已部署 LB 实例的集群主机的 CPU 插槽数之和。 |
| **过期时间** | 订阅许可/试用许可：精确到日，如 2021-05-01 永久许可：永不过期 | 试用许可默认 90 天，申请时最长可申请 180 天。 |

# 许可管理

## **首次部署后自动生成**

在 CloudTower 中**首次启用 LB 功能时**，默认生成**试用许可**。

* 版本：标准版  
* 许可类型：试用许可  
* 许可方式：按 vCPU 数  
* 许可过期时间：许可生成时间 \+ 90 天  
* 许可额度：不限

说明：首次生成的试用许可，仍默认按 vCPU 数计算，用户试用期间，也建议继续申请并使用按 vCPU 计算的许可，避免试用期间由于许可数不足阻碍集群扩容。

## **更新许可**

更新许可时，需检查如下信息

* 版本检查

  导入新版本许可时，必须满足 CloudTower 为 4.7.0 或以上版本，否则无法导入

* 额度检查

  更新为按 CPU 插槽数许可时，如许可点数（最大 CPU 插槽数）小于当前 CloudTower 中，所有 LB 实例所属集群中主机 CPU 插槽数之和，则不允许更新。

* 过期时间检查

  如许可过期时间早于当前时间，则不允许更新。

* 其他  
  * 许可类型：无限制，试用许可、永久许可、订阅许可间均支持相互转换。  
  * 许可方式：无限制，只要导入额度满足要求，两种方式间支持互相转换。

## **许可展示**

需展示当前许可方式，总额度、已使用额度和可用额度信息。

# 许可过期影响

过期提示内容、提示方式、过期后的影响操作均无变化。

# 商务系统签发许可

试用、正式及内部许可的申请中，申请**负载均衡器许可**时：

* 增加「许可方式」字段  
  * 选项为「按 vCPU 数」和「按 CPU 插槽数」；  
* 增加「CPU 插槽数」字段  
  * 许可方式为「按 CPU 插槽数」时，需填写此项；  
  * 填写整数，无范围限制；  
  * 许可方式为「按 vCPU 数」时，此项不展示，默认为无限。
