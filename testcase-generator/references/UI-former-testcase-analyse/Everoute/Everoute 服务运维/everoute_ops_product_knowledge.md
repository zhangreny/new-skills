# Everoute 服务运维产品知识摘要

## 适用范围

- 本文件用于 Step6-10 的运维知识加载，帮助识别 ER 部署、Controller 管理、节点替换、许可联动和异常限制的拆分方式。
- 运维场景常同时跨越安装与升级指南、发布说明和其他产品能力边界，本文件用于先建立版本/集群/节点骨架，再回查具体产品文档。

## 产品对象树

- 部署对象：部署 Everoute 服务、启用功能、ER 设置、系统服务 IP、管理网络。
- Controller 对象：Everoute Controller 工作节点、仲裁节点、所属集群、节点数量、虚拟机网络、管理 IP。
- 版本对象：ER 版本、Tower 版本、普通集群、跨集群、双活 5.x、双活 6.x。
- 运维动作对象：增加节点数量、编辑工作节点、替换工作节点、替换仲裁节点、恢复同步状态、扩缩容、许可更新。
- 许可对象：集群许可、节点许可、子功能许可、LB 物理 CPU 许可、跨产品许可文案。
- 固定块对象：事件审计与任务、权限管理、异常操作限制、文案提醒。

## 页面与动作树

- 部署链路：基本信息 -> Everoute Controller -> 分布式防火墙 -> VPC -> LB。
- Controller 部署链路：选择版本 -> 集群类型 -> 工作节点 -> 仲裁节点 -> 提交校验。
- ER 设置链路：详情页 -> 节点卡片 -> 增加节点数量 -> 编辑网络 -> 节点替换 -> 恢复状态。
- 许可链路：Tower 许可卡片 -> 部署/启用/创建/扩容场景 -> 额度判断 -> 跨产品文案变化。
- 固定块链路：事件审计与任务、权限管理、异常操作限制、灰色禁用原因、提示文案。

## 历史样本子标题习惯

- `Controller 跨集群` 样本通常按 `部署 ER 服务 / ER 设置 / 事件审计与任务 / 权限管理 / 异常操作限制` 拆顶层。
- 在部署或设置页内，样本会优先拆 `ER 版本 -> 集群类型 -> section -> 字段组/校验点 -> 结果`。
- 节点相关场景常继续按 `工作节点 / 仲裁节点 / 所属集群 / 节点数量 / 批量填充 / 提交校验` 拆分。
- `LB 支持物理 CPU 许可` 样本常按 `场景 -> 关联功能 -> 许可子类 -> 环境对象 -> 容量状态 -> 结果` 拆分。
- 样本里存在“复合点击”“批量填充”等复合 case 写法，这些只能作为拆分线索，不能原样照搬。

## 高频固定块

- 事件审计与任务：部署、编辑、替换节点、扩缩容、恢复状态、许可相关动作。
- 权限管理：具有编辑权限、仅可浏览、操作按钮显隐、是否允许提交。
- 异常操作限制：ER 异常、节点异常、集群连接异常、同步异常、可操作/不可操作边界。
- 版本与部署前提：ER 版本差异、Tower 版本差异、双活集群版本差异、仲裁节点可用性。

## 高优先级拆分轴

- 版本轴：ER 版本、Tower 版本、发布说明中的能力边界。
- 部署形态轴：单集群、跨集群、双活 5.x、双活 6.x。
- 节点轴：工作节点、仲裁节点、所属集群、节点数量、可用域。
- 表单行为轴：字段必填、禁用、过滤、默认值、批量填充、切换后清空、提交校验。
- 许可轴：许可对象、许可方式、额度状态、可部署/不可部署、可扩容/不可扩容、文案提醒。
- 异常轴：集群连接异常、节点异常、同步异常、灰色禁用原因、顶层 tip。
- 固定块轴：事件审计、权限管理、异常操作限制。

## 资源类型与前提

- 版本资源：ER 版本、Tower 版本、CloudTower 高可用状态、发布说明限制。
- 环境资源：普通集群、跨集群、双活集群、可用域、主机数、工作节点数、仲裁节点可用性。
- 对象资源：Controller 工作节点、仲裁节点、管理网络、系统服务 IP、LB 相关许可卡片。
- 许可资源：按集群、按节点、按子功能、按 CPU 插槽、按 vCPU、额度已用/可用、过期状态。
- 结果资源：部署成功、启用成功、创建成功、扩容成功、不可部署、不可扩容、文案变化。
- 典型前提：版本满足配套要求、安装包架构满足、集群过滤满足、仲裁节点存在且可用、功能启用顺序正确。

## 复合 case 拆分信号

- 单条 case 同时描述多个操作，如添加节点、批量填充、删除部分节点、手动修改 IP。
- 单条 case 同时覆盖版本差异、集群类型差异和节点类型差异。
- 单条 case 同时覆盖字段显隐、过滤逻辑、灰色禁用原因和提交结果。
- 单条 case 使用“多个”“批量”“复合点击”“反复切换”等词，且实际涉及多个独立动作。
- 单条 case 同时覆盖许可不足判断和跨产品文案归属。

## 禁止合并项

- 禁止把 ER 3.4 及以上与低于 3.4 的行为混写。
- 禁止把普通集群、跨集群、双活 5.x、双活 6.x 混成一个“集群类型”叶子。
- 禁止把 `字段必填 + 过滤 + 默认值 + 提交结果` 压成一条 case。
- 禁止把工作节点与仲裁节点的行为混写。
- 禁止在许可类场景里只看额度，不保留部署形态、集群关系、节点关系和结果归属。

## 术语与版本提醒

- `Everoute Controller` 的核心拆分通常围绕工作节点、仲裁节点、所属集群、节点数量和部署形态展开。
- 双活 6.x 和双活 5.x 的配置方式不同，尤其是可用域、节点数量和仲裁节点要求。
- 运维场景很多约束来自安装与升级指南和发布说明，而不是某个业务页面本身。
- 许可类运维需求经常跨产品，尤其会同时涉及 LB、VPC、DFW 的额度与文案归属。
- “异常操作限制”不是兜底分组，而是要针对具体异常状态和具体动作继续拆分。
