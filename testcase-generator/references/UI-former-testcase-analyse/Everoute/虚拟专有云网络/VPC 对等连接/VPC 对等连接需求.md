# **VPC 需求**

# 相关概念

| 中文 | 英文 | 定义说明 |
| ----- | ----- | ----- |
| **虚拟私有云** | **VPC, Virtual Private Cloud** | 是一个用户自定义的逻辑隔离网络空间。用户可以在 VPC 中管理自己的子网结构、IP 地址范围和分配方式、网关服务、安全服务等。 |
| **VPC 子网** | **VPC Subnet** | 指 VPC 内的子网。同一 VPC 子网内可以通信，同一 VPC 内的 VPC 子网间可以通信。 |
| **边缘网关** | **Edge Gateway** | 本需求中特指 VPC 服务的边缘网关虚拟机，它具有到 VPC 的下行链路连接，以及到物理网络的外部连接，作用是实现 VPC 与物理网络的三层互通。 如果 VPC 需要与数据中心、公网等外部网络进行通信，则必须先在服务中创建边缘网关。 |
| **/** | **Provider Network** | Provider Network 是和物理网络有直接映射关系的、可路由的虚拟网络（通常使用 VLAN）。Provider Network 由管理员创建，与租户/VPC 共享。 |
| **外部子网** | **External Subnet** | 是 Provider Network 的子网，用作外部可路由的 IP 资源池。外部子网中的 IP 地址可以用作 VPC 的 NAT IP、浮动 IP、LB  VIP 等。 |
| **安全组** | **Security Group** | 是安全策略应用对象的组合，VPC 的安全组只对关联了该 VPC 网络的虚拟机有效。可在安全组中添加标签组合作为动态成员，也可以添加指定虚拟机作为静态成员。如需对特定业务的虚拟机应用相同的安全策略，可以将其添加至同一个安全组中，并在各类所需策略中使用安全组作为策略对象。 |
| **安全策略** | **Security Policy** | 策略以携带对应标签或包含在对应安全组中的虚拟机为策略的目标对象，控制目标对象在出方向和入方向上的流量。VPC 的安全策略，在该 VPC 内有效。 |
| **网络地址转换** | **NAT, Network Address Translation** | SNAT, 源地址转换。当内部地址要访问外部网络，内部地址发起主动连接，SNAT 会将内部地址的私有 IP 转换为 SNAT IP（分配自外部子网）。主要用于内部共享 IP 访问外部。 |
|  |  | DNAT, 目标地址转换。当内部需要提供对外服务时，外部地址发起主动连接访问 DNAT IP（分配自外部子网），DNAT 接收这个连接后将连接转换到内部。主要用于内部服务对外发布。 |
| **浮动 IP** | **Floating IP** | 是一种 1:1 NAT。用户可以通过申请，把外部子网中的 IP 绑定给需要被外部主动连接的虚拟机，效果等同于公有云中的弹性公网 IP。 |
| **负载均衡器** | **LB, Load Balancer** | 负责将网络或应用流量负载平摊到多个后端服务器运行，提高应用容量和可靠性。 |
| **L2 网关** | **L2 Gateway** | 负责打通传统 VLAN 网络中的子网与 VPC 中的子网，将两个网络实现为一个二层子网，使传统网络的 VLAN 与 VPC 子网逻辑上在一个子网中。 |
| **VPC 对等连接** | **VPC Peering** | 是两个 VPC 之间的网络连接，可以实现两个 VPC 之间私网互通。 |
| **隧道端点** | **TEP, Tunnel Endpoint** | 每个传输节点（本文中指关联集群中的每个节点）具有一个隧道端点，负责将 Overlay 网络虚拟机流量进行封装，并将数据包路由到目标 TEP 以进行进一步的处理。TEP 是在外部 IP 标头中使用的源和目标 IP 地址，用于标识发起和结束 Overlay 网络帧的节点。 |

# 功能详情

## **VPC 对等连接（不在 1.0 实现）**

VPC 对等连接是两个 VPC 之间的网络连接，可以通过建立 VPC 对等连接且指定路由，实现两个 VPC 的 VPC 子网之间互通。

VPC 对等连接的使用规则：

* 不能跨 ER 服务创建对等连接，即两个 VPC 必须属于同一 ER 服务。  
* 建立对等连接的两个 VPC，如果它们的 VPC 子网存在地址重叠，以 VPC 内部路由优先。

在对等连接页面**提示**：需要在 2 个 VPC 的路由表中配置下一跳为对等连接的路由规则，对等连接才能生效。

### **创建**

创建对等连接的配置参数如下表所述：

| 配置参数 | 说明 | 示例 | 是否必填 |
| :---- | :---- | :---- | :---: |
| 名称 | 自定义命名对等连接。 | “VPC1-VPC2” | Y |
| 所属服务 | 选择一个开启了 VPC 功能的 Everoute 服务。 | / | Y |
| VPC 1 | 选择当前服务内的一个 VPC 作为对等连接的其中一端。 | / | Y |
| VPC 2 | 选择当前服务内的一个 VPC 作为对等连接的另一端端。 需要校验两个 VPC 不是同一个。 | / | Y |

![][image26]

#### **检查项**

如果两个 VPC ~~不~~是同一个，提交任务失败，ErrorMsg：不能连接同一个 VPC。

### **编辑**

可以编辑名称。不可更改所属服务、VPC 1、VPC 2。

### **删除**

删除需满足条件：两个 VPC 的路由表中均已删除下一跳为该对等连接实例的路由规则。

### **查看**

点击**网关服务\>对等连接**，可以查看所有 VPC 的对等连接。

点击 **VPC 列表与详情**，进入某个 VPC 的详情页面，选择**网关服务**\>**对等连接**，可以看到以该 VPC 作为其中一端的对等连接。

#### **列表页**

对等连接列表页面的字段说明如下：

| 字段名 | 说明 | 示例 | 默认列 | 可筛选 | 可排序 |
| :---- | :---- | :---- | :---: | :---: | :---: |
| 名称 | 对等连接的名称。 | “VPC-Peering-name” | Y | Y | Y |
| 所属服务 | 对等连接网关所属的 ER 服务。 | “ER-name” | Y | Y |  |
| VPC 1 | 对等连接其中一端的 VPC 名称。 | “VPC-name1” | Y | Y |  |
| VPC 2 | 对等连接另一端的 VPC 名称。 | “VPC-name2” | Y | Y |  |

#### **详情面板**

在列表中点击某个对等连接所在行，出现该对等连接的详情面板，可以查看以下信息：

* 详情

  展示所属服务、VPC1、VPC2。

* 关联路由表，展示关联路由表的名称、描述。
