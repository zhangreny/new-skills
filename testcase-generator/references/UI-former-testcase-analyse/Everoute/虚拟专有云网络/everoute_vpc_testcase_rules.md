# Everoute VPC UI Testcase Rules

## 必须遵守

- 顶层优先按“列表 / 创建 / 编辑 / 删除 / 详情 / 事件审计 / 其他相关改动 / 权限控制”组织。
- VPC 产品中，列表展示、详情页 tab、表单字段、路由或关联对象联动，必须保持独立分组，不允许混写。
- 只要详情页存在 tab 或附属对象（如路由表、对等连接、子网、TEP IP 等），必须先拆 tab/对象层，再落 case。
- 只要字段显隐、可编辑性、过滤范围、默认值、错误提示不同，就必须拆成字段级分组。
- 权限控制、事件审计、删除限制、关联限制属于 VPC 高频固定块，默认必须纳入。

## 常见分层模式

- 业务对象 -> 列表 / 创建 / 编辑 / 删除 / 详情 -> 字段或校验点 -> `[C]`
- 详情 -> tab -> 展示项或关联对象 -> `[C]`
- 业务对象 -> 事件审计 -> 动作类型 -> `[C]`
- 业务对象 -> 权限控制 -> 可编辑 / 只读 -> `[C]`
- 业务对象 -> 其他相关改动或限制 -> 关联对象/引用对象 -> `[C]`

## 展开规则

- 高优先级展开轴：页面动作、字段显隐、过滤逻辑、详情 tab、关联对象状态、权限、删除限制。
- 长名称展示、超长文案、链接跳转、详情页复用展示，通常应放在页面或 tab 分组下统一展开。
- 只要字段的前置选择会影响后续字段可选范围或禁用态，就必须把这种前后依赖关系显式展开。
- 对等连接、多网关、路由表、TEP IP 等带“关联对象”的场景，优先按对象关系展开，再看结果。
- 对文案调整类需求，必须单独保留“文案变化”结果轴，不要并入普通成功/失败场景。

## 禁止项

- 禁止把列表页、详情页、弹窗页的检查点混写在一个分组里。
- 禁止把“字段展示正确 / 字段可编辑 / 字段过滤正确 / 字段报错正确”压成一条 case。
- 禁止遗漏事件审计、权限控制、删除限制、关联限制这些 VPC 典型固定块。
- 禁止在有 tab、stepper 或关联对象时跳过中间层直接落 `[C]`。
- 禁止把具体路由、对等连接、TEP IP 的关联前提直接压缩成抽象“配置正确”。
