# Everoute 分布式防火墙产品知识摘要

## 适用范围

- 本文件用于 Step6-10 的 DFW 知识加载，帮助在历史样本不足的前提下，稳定识别 DFW 的对象树、策略结构、拆分轴和复合 case 信号。
- DFW 仍以 `everoute_common_testcase_rules.md` 为主骨架，本文件用于补齐 DFW 专属对象、术语和资源展开知识，不伪造黄金参考样本。

## 产品对象树

- 服务级对象：分布式防火墙概览、关联集群、许可、权限。
- 安全组对象：虚拟机安全组、IP 安全组、Pod 安全组。
- 策略对象：全局安全策略、自定义安全策略、虚拟机隔离策略、安全策略查询。
- 规则对象：策略对象、出入流量名单、允许出/入站流量规则、日志启用状态、监控模式、命中计数、Syslog 传输。
- 辅助对象：服务、标签、协议与端口、策略状态、导入导出、审计动作。
- 固定块对象：事件审计、权限、异常限制、联动结果、许可结果归属。

## 页面与动作树

- 集群链路：关联集群 -> 编辑或删除关联集群 -> 集群状态、过滤、许可与结果。
- 安全组链路：创建虚拟机安全组 / IP 安全组 / Pod 安全组 -> 编辑 -> 删除 -> 详情。
- 策略链路：设置全局安全策略 -> 创建放行策略 -> 创建拒绝策略 -> 编辑/删除/导入导出 -> 日志 -> 命中计数。
- 隔离链路：隔离单个虚拟机、批量隔离虚拟机、编辑隔离策略、停止隔离虚拟机。
- 观测链路：安全策略查询、日志下载与检索、Syslog 配置、命中计数通知与重置。

## 适合沿用的子标题习惯

- 顶层优先按对象或功能域拆：`关联集群 / 安全组 / 自定义安全策略 / 全局安全策略 / 隔离策略 / 查询与日志 / 固定块`。
- 第二层优先按页面或动作拆：`列表 / 创建 / 编辑 / 删除 / 详情 / 导入导出 / 日志 / 计数 / 权限 / 异常限制`。
- 第三层优先按策略类型或对象来源拆：`放行 / 拒绝 / 虚拟机 / IP / Pod / 标签 / 安全组 / 名单 / 规则状态`。
- 在联动或许可场景中，应继续按 `许可对象 / 许可方式 / 关联集群状态 / 结果归属` 拆分。

## 高频固定块

- 事件审计：关联集群、创建/编辑/删除安全组、创建/编辑/删除策略、启用日志、重置计数。
- 权限管理：只读/可编辑、是否允许创建或删除、是否允许导入导出。
- 异常限制：被策略引用的安全组不可删除、关联集群状态异常、策略对象不可选、日志或计数能力未满足版本前提。
- 联动结果：与 LB、VPC、运维联动时，报错只包含 DFW、文案只调整 DFW、许可结果只归属于 DFW 的场景必须保留。

## 高优先级拆分轴

- 策略类型：全局安全策略、自定义安全策略、隔离策略、查询、日志、命中计数。
- 自定义策略类型：放行策略、拒绝策略。
- 策略对象来源：虚拟机标签、虚拟机安全组、IP 安全组、Pod 安全组、IP 地址。
- 名单来源：白名单/黑名单、入流量/出流量、是否为源端配置允许出站流量规则。
- 对称策略：是否需要同时检查源端与目标端规则、是否属于自动补全的对称关系。
- 日志与观测：日志启用状态、监控模式、计数通知、Syslog、可观测性版本前提。
- 许可与集群：关联集群许可、按 CPU 插槽、按策略对象或 VM、额度状态、结果归属。
- 结果类型：显隐、筛选、禁用、校验报错、提交成功、文案变化、导入导出限制。

## 资源类型与前提

- 环境资源：ER 版本、CloudTower 版本、可观测性版本、SKS 版本、EIC CNI 插件状态、关联集群状态。
- 对象资源：安全组、服务、策略对象、白名单/黑名单来源、隔离虚拟机。
- 许可资源：关联集群许可、策略对象/VM 许可、额度状态、结果归属。
- 观测资源：日志功能、命中计数、Syslog、查询界面。
- 联动资源：与 LB 的许可文案、与 VPC 的许可文案、与运维的关联集群或版本前提。
- 典型前提：先启用 DFW、先关联集群、满足版本与插件要求、满足可观测性前提、相关安全组或服务已存在。

## 复合 case 拆分信号

- 单条 case 同时描述放行策略和拒绝策略。
- 单条 case 同时描述多种策略对象来源或多种名单来源。
- 单条 case 同时覆盖对象显隐、端口校验、规则状态和提交结果。
- 单条 case 使用“同时”“多个”“批量”“复合”时，实际覆盖多个对象类型、多个规则方向或多个结果。
- 单条 case 同时覆盖联动结果与 DFW 自身结果。
- 单条 case 同时覆盖日志启用、监控模式、命中计数或导出下载等多个观测能力。

## 禁止合并项

- 禁止把 `放行策略` 与 `拒绝策略` 混为一个叶子。
- 禁止把 `虚拟机安全组 / IP 安全组 / Pod 安全组` 混成“安全组创建正确”。
- 禁止把 `策略对象来源` 与 `白名单/黑名单来源` 压成一个字段维度。
- 禁止把 `日志 / 监控模式 / 命中计数 / Syslog` 合并为“观测功能正常”。
- 禁止把 `许可对象 + 许可方式 + 关联集群状态 + 结果归属` 缩成一条通用报错。
- 禁止在样本不足时凭空补出与当前需求无关的 DFW 产品矩阵。

## 术语与版本提醒

- DFW 常见高频术语包括：放行策略、拒绝策略、全局白名单、默认拒绝通信、对称策略、监控模式、命中计数。
- 虚拟机安全组、IP 安全组、Pod 安全组是三种不同的对象来源，成员规则和前提不同。
- Pod 相关策略受到 SKS 版本、EIC CNI 插件和控制平面连通性的约束，不能与虚拟机或 IP 场景混写。
- 当许可方式为按虚拟机数许可时，部分对象来源与配置能力会受限，必须保留为独立拆分轴。
- 可观测性版本会直接影响日志和命中计数能力，不能仅凭页面显隐判断。
