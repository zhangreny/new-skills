# Everoute 负载均衡 UI Testcase Rules

## 必须遵守

- Step6 先读取 `everoute_lb_product_knowledge.md`，用其确定对象树、常见子标题、固定块和高优先级拆分轴，再决定读取哪些 SmartX docs 与历史样本。
- 顶层优先按对象组织，例如 LB / LBI / LBG / 虚拟服务 / 服务器池 / 服务器池组 / 事件审计与任务 / 权限管理 / 异常限制。
- 第二层优先按页面或动作组织，例如部署 Everoute 服务、启用 LB、创建、编辑、详情、关联网络、解除关联、删除。
- 对象层次深的场景必须继续拆到“子模块 / 通信方向 / 网络类型 / 字段组 / 配置 IP / 状态”后再落 `[C]`。
- 只要版本、VPC 开关状态、网络模式、通信方向、主备一致性、池/池组类型不同导致展示或校验不同，就必须显式拆层。
- LB 相关需求通常有强矩阵展开，不允许把“对象 × 动作 × 版本/模式 × 结果”过早压缩成单条 case。
- 事件审计、权限管理、异常限制属于 LB 固定块，默认必须纳入。

## 常见分层模式

- 对象 -> 页面/动作 -> 子模块 -> 状态/条件 -> `[C]`
- 对象 -> 网络设置/关联网络 -> 通信方向 -> 字段组 -> 校验或结果 -> `[C]`
- 对象 -> 创建/编辑 -> 开关或模式 -> 网络类型/字段 -> `[C]`
- 对象 -> 详情 -> 卡片或 tab -> 展示项 -> `[C]`
- 对象 -> 事件审计与任务 / 权限管理 / 异常限制 -> 动作或限制类型 -> `[C]`

## 展开规则

- 高优先级展开轴：对象类型、动作类型、版本、VPC 支持状态、网络类型、通信方向、主备关系、容量状态、结果。
- 只要是网络配置类场景，优先按“网络类型 -> 字段组 -> 校验/分配方式”展开。
- 只要是主备场景，主实例、备实例、主备一致、主备不一致必须显式区分。
- 只要是关联网络/网络组场景，与客户端通信、与服务端通信、客户端+服务端通信应优先保留为中间层。
- 只要存在自动分配/手动分配/部分自动部分手动等模式差异，应在字段或配置组下继续展开。
- 许可类 LB 需求，优先使用“场景 / 关联功能 / 许可子类 / 环境对象 / 容量状态 / 结果”的组合模式。
- Step7/8 必须把历史样本中的“反复关联、取关”“同时关联”“多个 VPC / 多个 LBI”当作继续拆分的信号，而不是直接继承为单条 `[C]`。

## 测试资源类型

### LB 对象资源

- LB
- LBI
- LBG
- 虚拟服务
- 服务器池
- 服务器池组
- LB 虚拟机详情
- 关联网络 / 关联网络组

### LB 网络资源

- VLAN 网络
- VPC 子网
- VPC 网络组
- TEP 网络
- VIP / LIP
- 通信方向：与客户端通信 / 与服务端通信 / 同时通信
- 主备关系：主 / 备 / 主备一致 / 主备不一致
- 配置方式：自动分配 / 手动分配 / 部分自动部分手动 / 复用

### LB 许可资源

- 许可对象：LBI 所在集群、LB 虚拟机、LB 实例 / LBG
- 许可方式：按 vCPU、按 CPU 插槽
- 许可类型：试用 / 订阅 / 永久
- 额度状态：充足 / 刚好 / 不足 / 已满
- 过期状态：即将过期 / 已过期 / 永不过期
- 消耗动作：部署 ER、启用 LB、创建 LBI、集群扩容
- 归还动作：删除 LBI、集群缩容

### LB 权限与审计资源

- 权限角色
- 审计动作
- 异常限制

## 黄金参考候选

### `LBG & LB 支持 VPC`

- 仅在当前任务同时满足以下信号时作为黄金参考候选：
  - 主产品是 LB，补充产品是 VPC
  - 主对象集中在 `LBI / LBG / 虚拟服务 / 部署 Everoute 服务 / 启用 LB`
  - 材料中显式出现 `支持 VPC`、网络类型、通信方向、主备关系、VPC 资源过滤或 VPC 影响项
- 相对数量级默认参考带：`0.8x-1.2x`
- 结构对齐优先于措辞相似度和绝对数量
- 参考样本路径：
  - `references/UI-former-testcase-analyse/Everoute/负载均衡/LBG & LB 支持 VPC/测试用例-LBG & LB 支持 VPC.md`

## 资源展开提示

- LB 许可类需求必须显式保留 `许可对象 + 许可方式 + 许可类型 + 额度状态 + 过期状态 + 消耗/归还动作 + 结果`。
- LB/VPC 联动任务中，网络资源、VPC 资源、主备关系和许可资源必须同时保留，不能只剩“成功 / 失败”。
- 只要动作会消耗、占用或归还资源，就必须将这些资源规则前置到 Step6 和 Step7，而不是等到 Step9 才在 case 文案里补。

## 禁止项

- 禁止把 LBI、LBG、虚拟服务、服务器池等不同对象混在同一动作分组下。
- 禁止把开关显隐、字段显隐、字段校验、提交结果压成一条 case。
- 禁止把主备关系、通信方向、网络类型等关键轴折叠成“配置正确”。
- 禁止遗漏权限管理、事件审计与任务、异常限制。
- 禁止在长层级对象里为了节省篇幅而突然减少中间分组深度。
- 禁止把历史样本中带有“反复 / 同时 / 多个 / 复合”的 case 标题原样落到最终 testcase。
